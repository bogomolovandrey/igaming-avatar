## 1. Видеоаватар (Блок 4 демо)

**Статус:** отложено на после MVP (текст + аудио).
**Дата постановки:** 2026-04-23.

**Контекст:** в сценарии демо Блок 4 — wow-эффект через видеокружок, где Алекс отвечает видеосообщением на вопрос ведущего. В MVP фокусируемся на текстовом и голосовом взаимодействии; видео возвращаем после того, как базовый диалог работает стабильно.

**Варианты, которые рассматривали:**

- **A. Pre-rendered clips только для Блока 4.** Записать 2-3 коротких видеокружка с заранее известными ответами (например, «история про Лестер 2016»). Оператор триггерит нужный клип по ходу демо. Ноль технических рисков. Минус: нет ответа на произвольный вопрос клиента в блоке 4.

- **B. Real-time talking head через сервис.** HeyGen Streaming / D-ID / Tavus / Simli. LLM → TTS → видео-аватар лип-синхронно произносит реплику на любой ввод. Стоимость ~$0.10-0.50/мин, задержка 1-3 сек сверху. На русском лучше всего HeyGen и Tavus.

- **C. Hybrid: локальный lip-sync поверх idle-loop.** Записать бесшовный 10-секундный idle Алекса, прогонять Wav2Lip / MuseTalk / Sadtalker для липсинка под TTS. Работает на любой текст, бесплатно после setup, качество липсинка ниже коммерческих сервисов.

- **D. Real-time только в Блоке 4 (рекомендованный).** В блоках 1-3 аватар — голос + статическая/animated картинка. В Блоке 4 подключаем real-time сервис (вариант B) именно когда нужен видеоответ. Честная интерпретация сценария — в сценарии видео и позиционируется как фича Блока 4.

**Что надо будет решить при возврате:**

- Какой сервис: HeyGen / Tavus / D-ID / Simli — после тестов на русском с нашим LLM-пайплайном.
- Нужен ли видеоаватар в Блоке 1 (первое знакомство) тоже, или только Блок 4.
- Fallback, если сервис лёг в момент показа.

## 2. Динамический стейт букмекерского сайта

**Статус:** отложено после MVP.
**Дата постановки:** 2026-04-23.

**Контекст:** в MVP весь стейт (матчи, коэффициенты, игрок, бонусы) статичный — один TypeScript-файл, который читают и фоновая страница, и LLM-контекст. Это source of truth для обеих сторон. Операторская панель мутирует стейт при триггерах (например, прибавляет $520 к балансу при «крупном выигрыше»).

**Что откладываем:**

- **Реальная подтяжка коэффициентов** (Sportradar / Betradar / Pinnacle API).
- **Live-scoring матчей** (изменение тотала, голов в реальном времени).
- **Реальные балансы и история ставок** из бэкенда оператора.
- **Сегментация пользователей** (VIP / new / RG-flagged) на основе реальной CRM.

**При возврате нужно решить:**

- Где граница между «стейт локально в памяти Node-процесса» и «стейт в реальной БД/интеграции».
- Кеширование и частота опроса внешних API.
