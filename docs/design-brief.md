# Design Brief — Avatar AI Demo

**Дата:** 2026-04-23
**Адресат:** дизайнер(-ы), которые будут делать макеты
**Связанные документы:**
- Сценарий демо: `docs/Avatar_AI_Demo_Scenario.docx.md`
- Техдизайн: `docs/superpowers/specs/2026-04-23-avatar-ai-demo-design.md`
- Бэклог (видеоаватар и др.): `docs/backlog.md`

---

## 1. Контекст проекта

Разрабатываем демо-версию AI-компаньона «Алекс» для букмекерских платформ. Это веб-виджет, который висит поверх сайта букмекера и общается с игроком текстом и голосом. Алекс ведёт онбординг, рекомендует ставки, эмоционально реагирует на выигрыши/проигрыши, мягко переключается в Responsible Gaming режим при проблемном поведении.

**Цель демо:** показать потенциальному корпоративному клиенту (букмекеру), что это такое, как это выглядит и ощущается, в том числе на реальных устройствах (мобильный, десктоп).

**Хронометраж демо:** 8-10 минут, 4 блока (знакомство, аналитика, триггеры + RG, видеоаватар). Блок 4 (видео) сейчас **вне scope** — делаем только текст + голос.

**Характер Алекса:** AI-компаньон, не «бот поддержки». Молодой парень (~25 лет), разбирается в спорте, дружеский тон, самоирония, уместные эмодзи, свои мнения. Эмоциональная связь — ключевая ценность.

---

## 2. Платформы и устройства

**Обязательно поддержать:**
- Десктоп (≥ 1024px)
- Мобильный (iOS Safari, Android Chrome) — 320-430px шириной в основном
- Планшет (768-1023px) — либо mobile-layout, либо desktop-layout, на усмотрение дизайнера

Showcase идёт с реального домена, так что макеты должны выдерживать реальное разрешение устройств.

---

## 3. Общая стилистика

**Направление — Dark & Serious** (референсы — Bet365, Stake, Pinnacle).

Плотная, строгая, «взрослая» букмекерская эстетика:
- Тёмно-синий / почти чёрный фон (`#0a0e14`, `#111827` и соседи)
- Золото / янтарь как акцент на коэффициентах (`#fbbf24` / подобные)
- Монохромные плашки матчей, высокая информационная плотность
- Шрифт — system-ui / Inter / Manrope, хорошо читается на мелком
- Круглые углы умеренные (4-8px для карточек, 20px для pill-элементов)

**Не делать:**
- Яркие казино-градиенты (фиолетовые, розовые) — это отдельное направление, его отвергли
- Светлую fintech-эстетику (DraftKings-style) — тоже отвергли
- Избыточные тени и blur — максимум тонкий subtle shadow у поднятых элементов (виджет)

Виджет Алекса может (и должен) мягко выделяться на фоне — это «живая» сущность поверх плотной страницы. Можно использовать градиент в аватарке, лёгкий accent color, отличный от золотого (например, индиго-розовый градиент для круглой аватарки, чтобы читалось как «живая личность»).

## 4. Экраны и состояния

### 4.1 Букмекерская главная страница (backdrop)

Это **фон**, на котором живёт виджет. Статичная страница, имитирующая реальный сайт букмекера. Не должна быть супер-проработанной — её задача: быстро дать понять «это букмекер», чтобы клиент сфокусировался на виджете.

**Компоненты страницы:**

- **Header (sticky):**
  - Логотип букмекера — условное имя `BETARENA`, шрифт bold, золотой акцент
  - Основная навигация: Спорт / Live / Казино / Промо
  - Правая часть: текущий баланс игрока (золотым), кнопка `+ Депозит` (золотая заливка или обводка), кнопка `⚙ Panel` (для открытия admin-view виджета)
  - Мелкий `session chip` в углу (см. 4.6)

- **Левый sidebar (desktop):**
  - Список «Топ лиги» с иконками: ⚽ Ла Лига, ⚽ АПЛ, 🏀 NBA + ещё 3-5 плейсхолдеров
  - Высота по содержимому, прилипает к странице при скролле

- **Центральный feed:**
  - Подзаголовок «Сегодня • ТОП матчи» с золотым акцентом
  - Карточки матчей — 3-5 штук, каждая содержит:
    - Название лиги (мелко, тускло)
    - Home vs Away команды (bold)
    - Время старта (тускло, справа)
    - Ряд с 2-3 коэффициентами (П1 / X / П2 для футбола; П1 / П2 для баскета), каждый в отдельной плашке с золотой цифрой
  - При наведении на карточку — лёгкий подъём (subtle transform)

- **Правый sidebar (desktop):**
  - Можно оставить пустым с небольшим промо-баннером или не включать (зависит от дизайнера)

**Мобильная версия страницы:**
- Header той же структуры, но компактный (гамбургер для навигации)
- Sidebar'ов нет, сразу центральный feed
- Карточки матчей одной колонкой

**Матчи в макетах должны содержать реальные данные из сценария:**
- ⚽ Реал Мадрид — Барселона (Ла Лига, 21:00), коэф 2.10 / 3.40 / 3.20
- ⚽ Ливерпуль — Манчестер Сити (АПЛ, 18:30), коэф 1.85 / 3.60 / 4.20
- 🏀 Лейкерс — Бостон (NBA, 03:00), коэф (примерно) 1.95 / 1.85

---

### 4.2 Виджет Алекса — свёрнутое состояние

**И на мобильном, и на десктопе.** Это — основное состояние, в котором виджет проводит бóльшую часть времени, когда пользователь не в чате.

Визуальная концепция — **«живой персонаж в углу»** (не маленький chat-bubble, а выразительный круглый аватар):

- Большая круглая аватарка Алекса — 64-80px, в правом нижнем углу страницы
- Градиентная обводка или фон аватарки (например, индиго→розовый) — чтобы аватарка «светилась» на тёмном фоне
- Маленький зелёный online-индикатор в правом нижнем углу аватарки, с белой обводкой
- Лёгкий continuous pulse-эффект при активной сессии (очень деликатный, не раздражающий)
- При получении proactive-реплики (от триггера) — над аватаркой появляется **speech bubble** с превью сообщения, плавно fade-in + subtle bounce, закрывается авто через 5-7 секунд или по тапу (открывает полный чат)
- Клик по аватарке → открывается полный виджет (см. 4.3 и 4.4)

Speech bubble:
- Максимум 2-3 строки превью
- Серый фон с backdrop-blur (полупрозрачный)
- Округлённый угол снизу-справа (хвостик bubble указывает на аватар)
- Белый текст

---

### 4.3 Виджет Алекса — развёрнутый **на десктопе**

**Паттерн — выезжающий правый sidebar** шириной ~400px. Букмекерская страница остаётся видимой слева — не overlay, а side-by-side. Это принципиально: «Алекс всегда рядом», а не «Алекс перекрывает всё».

Структура sidebar'а сверху вниз:

1. **Заголовок (~80-100px):**
   - Большая аватарка Алекса по центру или слева (40-48px)
   - Имя `Алекс` + маленький статус `● онлайн` (зелёным)
   - Справа — кнопка закрытия виджета (×), чтобы вернуться в свёрнутое состояние
   - Subtle gradient-фон для выделения heading из остального тела

2. **Tabs — переключатель между Chat и Admin:**
   - Два таба: `Chat` (default) и `⚙ Admin`
   - Активный — подсвечен золотым
   - Неактивный — приглушённым текстом

3. **Контент выбранного таба (см. 4.5 и 4.6)**

4. **Input-bar (зафиксирован внизу sidebar'а):**
   - Поле ввода текста с placeholder «Напиши Алексу…»
   - Кнопка-микрофон справа — круглая, золотая или accent-color заливка. При зажатии — меняется цвет, начинается анимация волны, показывается live-транскрипт в input-поле

---

### 4.4 Виджет Алекса — развёрнутый **на мобильном**

**Fullscreen takeover.** Открывается плавным slide-up снизу либо fade-in поверх всей страницы.

Структура сверху вниз:

1. **Хедер виджета (sticky, ~60-70px):**
   - Кнопка ↓ (свернуть) слева — возвращает в свёрнутое состояние
   - По центру: мини-аватарка + имя `Алекс` + статус
   - Справа: кнопка ⋯ (меню — на будущее, может быть пустой сейчас)

2. **Tabs** (`Chat` / `Admin`) — теми же правилами что на десктопе

3. **Контент** таба (см. 4.5 / 4.6)

4. **Input-bar** зафиксирован у нижнего края экрана, над системной клавиатурой

---

### 4.5 Chat-вид (внутри виджета, обе платформы)

**Структура сверху вниз:**

- **Блок с аватаром и именем Алекса** (если его нет в заголовке виджета, то здесь)
- **Scroll-область с сообщениями:**
  - Сообщения Алекса — слева, в серых bubble'ах, с хвостиком слева-снизу, белый текст
  - Сообщения пользователя — справа, в золотой или accent-color bubble'ах, белый текст
  - System events (например, notification «Алекс инициирует разговор» или «Триггер сработал» — **не показывать пользователю**, это для админа)
  - Между сообщениями Алекса и пользователя — адекватный padding, хорошо читается
  - Время мелкое, под bubble'ом или только при долгом hold
- **Typing indicator** — три точки в сером bubble'е, когда Алекс «печатает» (стримит токены из LLM)
- **Speech playback indicator** — когда Алекс говорит вслух, визуальная полоска-progress или волна под его последним сообщением. Можно тапнуть, чтобы прервать озвучку
- **Interim transcription** — когда пользователь зажимает микрофон и говорит, в input-поле появляется live-текст распознаваемой речи (светло-серым)

---

### 4.6 Admin-вид (внутри виджета, обе платформы)

Это **не скрыто** от клиента — ведущий сам открывает его на демо и объясняет «в продакшне триггеры приходят из вашего CRM, но для демо мы это вынесли сюда». Нужно сделать явно «инженерно-техническим», чтобы не было двусмысленности.

**Стилистика admin-вида отличается от chat-вида:**
- Более техничный, менее «персонажный»
- Тот же тёмный фон, но слегка другой оттенок для визуальной разделки от chat
- Моноширинный шрифт для технических полей допустим

**Структура сверху вниз:**

1. **Session Info** — маленький блок:
   - Session ID (формат `bright-fox`)
   - Текущий mode (`normal`, `celebration`, `rg_care`, `bonus_nudge`)
   - Баланс, loginCount

2. **Triggers — Preset** (основная часть):
   - 5 больших кнопок в вертикальной column:
     - 🎉 **Крупный выигрыш** — `big_win`
     - 😔 **Серия проигрышей (5x)** — `loss_streak`
     - 🎁 **Фрибет сгорает (48ч)** — `freebet_expiring`
     - 🎭 **Показать галерею персонажей** — `show_gallery`
     - 🔄 **Сбросить сессию / первый визит** — `first_visit`
   - Каждая кнопка — прямоугольная, 48-56px высотой, с иконкой слева и названием триггера, опционально — подпись меньшим шрифтом с описанием эффекта
   - При нажатии — короткая подтверждающая анимация (отмечается активация)

3. **Custom Event** (свёрнутый блок):
   - Заголовок `📝 Custom event`
   - Text area для описания события (placeholder: «Игрок только что пополнил $500»)
   - Dropdown для смены mode опционально
   - Кнопка `Отправить`

4. **Inject Message** (свёрнутый блок):
   - Заголовок `💬 Inject as user`
   - Text area: «Реплика от имени пользователя»
   - Кнопка `Отправить` + галочка «озвучить голосом» (если активна voice-сессия)

5. **Mini-transcript** (нижняя часть):
   - Последние 10 реплик в компактном виде
   - Имя + текст, монохромно, читабельно но не отвлекает
   - Scroll, если не влезает

### 4.7 Галерея персонажей

Открывается либо через триггер `show_gallery` из admin (во время демо), либо через кнопку в chat-вью Алекса (ведущий может попросить «покажи кто ещё у вас есть»).

**На десктопе** — модальное окно по центру, ~800x600px, над затемнённым фоном.
**На мобильном** — fullscreen view с кнопкой закрытия сверху.

**Содержимое:**
- Заголовок: `Твоя команда компаньонов`
- Сетка 3x3 из 9 карточек персонажей (или 2x5 на мобильном)
- Каждая карточка:
  - Квадратный портрет персонажа (60-80% высоты карточки)
  - Имя персонажа под портретом
  - Короткое описание (1 строка) — «Аналитик», «Эмоциональная девушка», и т.д.
- **Алекс** — активен, полноцветный, подписан `● Активен` (зелёным)
- **Остальные 8** — серый фильтр (`grayscale(0.7)`), оверлей с текстом `Coming soon`, слегка затемнены. При hover — тултип «Скоро»

**Персонажи (визуальные архетипы для генерации портретов):**

1. **Алекс** (активен) — молодой парень ~25, уверенная улыбка, casual-стиль, тёплая подача
2. Макс — ботаник-аналитик в очках, педантичный
3. Лика — молодая девушка-тусовщица, эмоциональная, яркая
4. Геннадий — ветеран-тренер, 50+, спокойный, умудрённый
5. Стас — стример-шутник, молодой, в кепке, энергичный
6. Вера — строгая дама-статистик в деловом стиле
7. Артём — хипстер-крипта, с бородой и тату
8. Борис — старая школа, 40+, мужественный, рыбак/охотник
9. Дрон — «bro-tier», спортивный, атлетичный

Это базовые архетипы — дизайнер может скорректировать, оставляя разнообразие.

**Портреты будут сгенерированы через Google Nano Banana 2 (Gemini 3.1 Flash Image)** — скрипт выполняется 1 раз, результат коммитится в репо. Дизайнер может дать prompt-идеи для каждого персонажа или подтвердить архетипы.

### 4.8 Session Chip

Маленький визуальный идентификатор сессии, видимый **ведущему на демо** (чтобы он понимал, что его действия применяются к правильной сессии).

- Размер: ~100-120px шириной, ~20px высотой
- Позиция: верхний правый угол страницы, под header'ом, или фиксирован внизу-слева
- Содержимое: `sess: bright-fox` маленьким моноширинным шрифтом, тусклым цветом
- Visible но не отвлекает
- При клике — копирует session ID в буфер (необязательно, приятно)

## 5. Ключевые анимации и microinteractions

### 5.1 Proactive Speech Bubble (над свёрнутой аватаркой)

Активируется при триггерном событии (big_win, loss_streak, freebet_expiring).

- Появление: **fade-in + subtle scale 0.9 → 1.0** за 300-400мс
- Лёгкий bounce в конце
- Исчезновение через 5-7 секунд ИЛИ при тапе по аватарке (открывает чат)
- Важно: не раздражающее появление, но заметное

### 5.2 Confetti при big_win

Когда срабатывает триггер `big_win`:
- По всей странице разлетается конфетти (золотые + accent-color частицы)
- Длительность ~2-3 секунды
- Одновременно в header'е баланс **тикает** до нового значения (`$840.00` → `$1360.00`) с анимацией счётчика ~1.5с
- Одновременно в чате Alex пишет proactive-реплику

### 5.3 RG Care визуальный сдвиг

Когда срабатывает `loss_streak` и mode переключается в `rg_care`:
- Виджет получает мягкий **warm-glow** по краям (тёплый янтарный/оранжевый, subtle)
- Или: edges виджета становятся округлее, фон чуть светлее, ощущение «более тёплого объятия»
- Аватарка Алекса — лёгкий smile-indicator или мягкое свечение
- Цель: визуально отличить RG-режим, чтобы клиент на демо видел «это другой режим заботы»
- Возврат в `normal` mode — плавный fade обратно

### 5.4 Freebet bonus nudge

Когда срабатывает `freebet_expiring`:
- Маленький badge с суммой фрибета появляется рядом с балансом в header'е
- Лёгкое pulse-animation на badge 2-3 раза, потом статичен
- Alex пишет proactive-реплику

### 5.5 Микрофон и голос

- При зажатии mic-кнопки — кнопка «пульсирует» accent-color'ом, вокруг появляется ripple или анимированная волна
- В input-поле — live interim transcription (светло-серым, обновляется по ходу распознавания)
- Когда Alex говорит вслух — под его последним сообщением появляется waveform / progress bar, можно тапнуть для прерывания

### 5.6 Typing indicator

Три точки пульсируют в сером bubble'е, пока Claude стримит ответ. Исчезает, когда streaming закончен.

### 5.7 Transitions

- Открытие виджета на desktop (slide-in right sidebar) — 300мс ease-out
- Открытие виджета на mobile (slide-up fullscreen) — 300мс ease-out
- Переключение Chat ↔ Admin — fast fade ~150мс
- Hover на кнопки триггеров — subtle scale/brightness

## 6. Типография

Предварительные направления, дизайнер уточняет:

- **Шрифт:** Inter / Manrope / system-ui — geometric sans, хорошо читается
- **Иерархия:**
  - H1 / хедеры — 18-24px, weight 700
  - Body / чат — 14-16px, weight 400-500
  - Captions / metadata — 11-12px, weight 400, приглушённый цвет
  - Коэффициенты — 14-16px, weight 600, золотой
  - Моноширинный (для session chip, технических полей в admin) — JetBrains Mono / ui-monospace

## 8. Что нужно от дизайнера (deliverables)

1. **Bookmaker backdrop** — десктоп + мобильный
2. **Widget collapsed** (floating avatar) — десктоп + мобильный (одинаково почти, различия в позиционировании если есть)
3. **Widget expanded — desktop sidebar** — состояния: Chat пустой / Chat с несколькими сообщениями / Chat с typing indicator / Chat с voice playback
4. **Widget expanded — mobile fullscreen** — те же состояния что desktop
5. **Admin view — desktop sidebar** — все preset-триггеры видны, custom event и inject-message блоки
6. **Admin view — mobile fullscreen**
7. **Character gallery — desktop (modal)** — 9 карточек, 1 активна, 8 coming-soon
8. **Character gallery — mobile (fullscreen)**
9. **Microinteraction specs** для ключевых анимаций (5.1-5.7) — короткое описание timing / easing, или motion-референсы (Lottie / Figma Smart Animate)
10. **Промо-варианты** (опционально) — если есть идеи как сделать фон букмекера отличным на 5-10% от Bet365, не теряя «серьёзности» — предложи

**Формат:** Figma-файл или аналог. Для видимости проекту — link share на read.

## 9. Что НЕ нужно (вне scope)

- Видеоаватар и связанный экран (Блок 4 сценария) — отложено
- Настройки / профиль пользователя — вне scope демо
- Депозит / кассовые экраны — кнопка есть на фоне, но открывает ничего
- Live-трансляция матча — нет
- Другие страницы букмекера (казино, live, промо) — есть только в навигации, не открываются
- Мультиязычность — только русский интерфейс
- Темизация (светлая тема) — только тёмная
