# **Технические заметки для разработки демки**

* **Скорость ответа:** максимум 2-3 секунды на ответ. Для демки допустимо использовать заготовленные ответы с имитацией набора текста (typing indicator)

* **Голосовой ввод:** обязателен fallback на текстовый ввод. Если распознавание подведёт в момент показа — переключиться на текст без потери темпа

* **Триггеры:** события победы/поражения/бонусов запускаются вручную через скрытую панель управления (не через реальную интеграцию с платформой)

* **Видеоаватар:** допустимо использовать pre-rendered видео, запускаемое по триггеру, вместо генерации в реальном времени

* **Фон платформы:** для демки использовать один фон (букмекерский). Не переключать между слотами и букмекером — это удваивает работу без добавления ценности

* **Галерея персонажей:** 8 неактивных персонажей — достаточно аватарок с лейблом «Coming soon». Функциональность не нужна, только визуал

* **Данные матчей:** для демки использовать реальные матчи ближайших дней. Обновить данные за 1 день до показа
